# TEST-1775763304 — Implement typed LRU cache module with TTL eviction and full test suite

Implement a production-quality LRU (Least-Recently-Used) cache module in TypeScript with TTL (time-to-live) support and a comprehensive Vitest test suite. The implementation uses a doubly-linked list combined with a Map to achieve O(1) time complexity for both get and set operations. The module is organized into type definitions, core implementation, and comprehensive test coverage.

## Wave 1

- [x] 1. Create `src/cache/types.ts` with TypeScript interfaces: `CacheOptions` (capacity, defaultTtlMs), `CacheEntry<V>` (value, expiresAt), and `CacheStats` (size, capacity, hits, misses, evictions, expirations). All interfaces require JSDoc comments and must avoid `any` types.
- [x] 2. Create `src/cache/lru-cache.ts` implementing `class LRUCache<K, V>` with a doubly-linked list + Map internal structure for O(1) operations. Implement constructor, get(), set(), has(), delete(), clear(), size getter, capacity getter, stats(), keys(), values(), entries(). All public API methods require JSDoc comments.
- [x] 3. Create `src/cache/index.ts` to re-export all public types and the LRUCache class from the module.
- [x] 4. Update `src/index.ts` to re-export from `src/cache/index.ts`, exposing LRUCache from the package root.
- [x] 5. Run `pnpm build` to verify TypeScript compiles without errors, then run `pnpm lint` to ensure no type issues.

## Wave 2

- [x] 1. Create `src/cache/lru-cache.test.ts` with a comprehensive Vitest test suite targeting 30+ test cases covering: basic set/get/has/delete/clear operations, LRU eviction order correctness, access updates recency (get moves entry to most-recent), TTL expiry behavior, default TTL option, stats accuracy for hits/misses/evictions/expirations, capacity=1 edge case, update existing key does not evict, delete removes entries from size, keys/values/entries return items in LRU order, and generic type support with different key/value types.
- [x] 2. Run `pnpm test` to verify all 30+ tests pass and achieve full coverage of the public API surface.
